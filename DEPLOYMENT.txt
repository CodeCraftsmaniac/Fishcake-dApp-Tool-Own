FISHCAKE DAPP - PRODUCTION DEPLOYMENT
=====================================

ARCHITECTURE:
-------------
Frontend: Vercel (Next.js)
Backend: Railway (Node.js/Express)
Database: SQLite (Railway persistent storage)
Blockchain: Polygon Mainnet

LIVE URLS:
----------
Frontend: https://fishcake-dapp.vercel.app
Backend API: https://fishcake-dapp-tool-production.up.railway.app
Health Check: https://fishcake-dapp-tool-production.up.railway.app/health

✅ Frontend: Auto-deploys from GitHub (main branch)

✅ Database: Persistent storage on Railway
✅ CORS: Configured for all Vercel URLs

ENVIRONMENT VARIABLES (Railway):
--------------------------------
NODE_ENV=production
PORT=8080 (auto-assigned by Railway)
FRONTEND_URLS=https://fishcake-dapp.vercel.app,...
DATABASE_PATH=/app/data/fishcake.db
MINING_DB_PATH=/app/data/mining.db
JWT_SECRET=<random-string>
RPC_PUBLICNODE=https://polygon-bor-rpc.publicnode.com
RPC_ANKR=https://rpc.ankr.com/polygon
SCHEDULER_ENABLED=true
LOG_LEVEL=info

MONITORING:
-----------
Backend Health: curl https://fishcake-dapp-tool-production.up.railway.app/health
RPC Status: curl https://fishcake-dapp-tool-production.up.railway.app/api/mining/rpc/status
Mining Status: curl https://fishcake-dapp-tool-production.up.railway.app/api/mining/status

DEPLOYMENT PROCESS:
-------------------
1. Push to GitHub main branch
2. Railway auto-detects changes
3. Builds backend (npm install && npm run build)
4. Deploys to production
5. Health check passes
6. Traffic routed to new deployment

MAINTENANCE:
------------
- Monitor Railway dashboard for errors

TROUBLESHOOTING:
----------------
If backend is down:
1. Check Railway dashboard for errors
2. Check health endpoint
3. Review deployment logs
4. Verify environment variables
5. Check RPC endpoints status

If frontend can't connect:
1. Verify NEXT_PUBLIC_API_URL is set
2. Check CORS configuration
3. Test backend health endpoint
4. Check browser console for errors
